# Compliance Frameworks

## Introduction

Compliance is the demonstration that your architecture and operations meet specific regulatory requirements. Azure provides a broad set of compliance certifications and tools to help organizations meet their obligations across jurisdictions.

## Topics to Cover

- [ ] Key compliance frameworks relevant to hybrid/sovereign architectures:
  - **GDPR** (General Data Protection Regulation) — EU
  - **NIS2 Directive** — EU cybersecurity
  - **DORA** (Digital Operational Resilience Act) — EU financial services
  - **FedRAMP** — US federal
  - **ISO 27001/27017/27018** — International
  - **SOC 1/2/3** — Service Organization Controls
  - **C5** — Germany (BSI Cloud Computing Compliance)
  - **ENS** — Spain (Esquema Nacional de Seguridad)
- [ ] Azure compliance certifications by region and service
- [ ] Microsoft Defender for Cloud regulatory compliance dashboard
- [ ] Azure Policy for compliance: built-in initiatives for regulatory standards
- [ ] Compliance in hybrid environments: bridging cloud and on-premises compliance
- [ ] Audit and evidence collection: Azure Activity Logs, Microsoft Purview Audit
- [ ] Shared responsibility model for compliance across the hybrid continuum
